hs.screen

Manipulate screens (i.e. monitors)

You usually get a screen through a window (see hs.window.screen). But you can get screens by themselves through this module, albeit not in any defined/useful order.

Hammerspoon's coordinate system assumes a grid that is the union of every screen's rect (see hs.screen.fullFrame).

Every window's position (i.e. topleft) and size are relative to this grid, and they're usually within the grid. A window that's semi-offscreen only intersects the grid.

API Overview

  • Functions - API calls offered directly by the extension
    • setPrimary
    • setTint
  • Constructors - API calls which return an object, typically one that offers API methods
    • allScreens
    • mainScreen
  • Methods - API calls which can only be made on an object returned by a constructor
    • frame
    • fullFrame
    • id
    • name
    • next
    • previous
    • toEast
    • toNorth
    • toSouth
    • toWest

API Documentation

Functions

setPrimary

Signature hs.screen:setPrimary(screen) -> nil
Type Function
Description

Sets the screen to be the primary display (i.e. contain the menubar and dock)

setTint

Signature hs.screen.setTint(redarray, greenarray, bluearray)
Type Function
Description

Set the tint on a screen; experimental.

Constructors

allScreens

Signature hs.screen.allScreens() -> screen[]
Type Constructor
Description

Returns all the screens there are.

mainScreen

Signature hs.screen.mainScreen() -> screen
Type Constructor
Description

Returns the 'main' screen, i.e. the one containing the currently focused window.

Methods

frame

Signature hs.screen:frame() -> rect
Type Method
Description

Returns the screen's rect in absolute coordinates, without the dock or menu.

fullFrame

Signature hs.screen:fullFrame() -> rect
Type Method
Description

Returns the screen's rect in absolute coordinates, including the dock and menu.

id

Signature hs.screen:id(screen) -> number
Type Method
Description

name

Signature hs.screen:name(screen) -> string

Description

next

Signature hs.screen:next() -> screen
Type Method
Description

Returns the screen 'after' this one (I have no idea how they're ordered though); this method wraps around to the first screen.

previous

Signature hs.screen:previous() -> screen

Description

Returns the screen 'before' this one (I have no idea how they're ordered though); this method wraps around to the last screen.
